PR 前自動衝刺

高手

讓 ECC 把 PR 前最後一公里自己跑完，不用你手動補 checklist

為什麼實用

很多任務其實主功能已經做好，但 PR 還不能送。真正卡住的是測試、文件、review、摘要這些收尾工。這個模式就是把收尾工自動化。

衝刺順序

1
先跑硬驗證
lint、type-check、test、build 先過
2
再跑軟驗證
code review、安全風險、命名與結構補漏
3
補文件與說明
README、變更說明、migration note 一起補齊
4
列殘餘風險
誠實寫出還沒涵蓋的邊界與 follow-up
5
最後整理 PR 摘要
直接產出可貼上的變更摘要與驗證結果

PR 前衝刺 Prompt

適合你已經快完成，但不想再自己扮演 checklist 機器人的時候
> 請幫我做一輪 PR 前自動衝刺，不要只說功能已完成。 請你自己完成： 1. 跑 lint / type-check / test / build 2. 再做一輪 code review 與安全性自查 3. 發現缺漏就直接補上 4. 補 README、使用說明或變更摘要（若有需要） 5. 最後給我一份可直接放進 PR 的摘要 如果任何一關失敗，先修好再繼續，不要中途停下來等我指示。

🚨 不要只要求『幫我整理 PR 描述』

如果前面的驗證沒跑，它只會幫你寫出一份漂亮但不可靠的 PR 摘要。先把 lint、test、build、review 收進同一輪衝刺才有意義。

最後輸出最好包含

1. 變更摘要。

2. 驗證結果與具體指令。

3. 主動補上的缺口。

4. 剩餘風險與 follow-up。

搭配建議

和評審迴圈直接串起來
> PR 前自動衝刺完成後，再交給 reviewer 打分；低於 8 分就再跑一輪補漏
