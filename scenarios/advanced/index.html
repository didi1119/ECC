多 Agent 並行作戰

📖 情境故事

技術主管阿強接到重構任務：同時要升級前端框架、重寫後端 API、遷移資料庫。 他用 /multi-plan 把任務拆給三個並行 Agent 同時執行， 原本需要 18 小時的工作，8 小時內全部完成。

ECC 的 38 個 Agents

每個 Agent 都是專門化的 AI，只負責自己擅長的領域。以下是最常用的 8 個：

planner規劃任務，等確認才動手/plan
code-reviewer審查程式碼品質與安全性/code-review
tdd-guide測試驅動開發引導/tdd
build-error-resolver自動修復建置錯誤/build-fix
security-reviewerOWASP 安全漏洞掃描自動觸發
architect系統架構設計決策自動觸發
doc-updater自動更新文件/docs
refactor-cleaner清理死碼與重構/refactor-clean

/multi-plan 多 Agent 並行

/multi-plan 自動拆分大任務， 啟動多個 Agent 並行執行，大幅縮短時間

情境示範 — 電商平台重構
/multi-plan 重構整個電商平台：前端 React 升級 + 後端 API 重寫 + 資料庫遷移

為什麼並行這麼重要？

❌ 串行執行（慢）

前端重構 → 6h
後端重寫 → 8h
資料庫遷移 → 4h

總計：18 小時

✅ 並行執行（快）

前端重構 → 6h
後端重寫 → 8h
資料庫遷移 → 4h

總計：8 小時節省 55%

進階指令

拆分任務給多個並行 Agent
> /multi-plan 你的大型任務描述
手動編排 Agent 執行順序
> /orchestrate
同時執行多個獨立任務
> /multi-execute
啟動完整開發艦隊（多 Agent 協作）
> /devfleet

什麼時候用多 Agent？

✓ 用多 Agent

大型重構（前後端同時改）多模組並行開發跨語言專案（TypeScript + Python）

✓ 用單 Agent

單一功能開發小 bug 修復文件更新
