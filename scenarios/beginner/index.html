新手必知技巧

✓ /plan 的核心價值：它不會立刻動手寫程式

很多人一開始誤以為 /plan 就是要讓 Claude 開始寫，其實它會先列出計劃等你確認，避免方向跑偏。

✓ 回覆 modify: xxx 可以調整計劃

不用重新說一遍需求，直接說「modify: 改用 Vue 而不是 React」，Claude 會更新計劃。

✓ /code-review 每次完成功能後都要跑

養成好習慣，寫完一個功能就 /code-review，比等 PR 被打回來再改省事多了。
