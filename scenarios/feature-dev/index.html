📖 情境故事

阿偉接到需求：「本週五前要上線購物車功能」。他用 /plan 把需求拆解成具體步驟， 再用 /tdd 先寫測試再實作， 中途遇到 TypeScript 錯誤，一鍵 /build-fix 搞定。 週四下午就提交 PR 了。

什麼是 TDD？

🔴 RED
先寫測試
測試會 FAIL，因為還沒實作
🟢 GREEN
最小實作
讓測試通過，不多不少
🔵 IMPROVE
重構優化
保持測試通過的同時改善程式碼

ECC 的 /tdd 指令會自動引導你完成這三個階段

完整開發流程

1
用 /plan 拆解需求
把需求告訴 Claude，它會拆成具體的實作步驟並等你確認
/plan 新增購物車功能
2
用 /tdd 開始測試驅動開發
先寫測試（RED）→ 再實作（GREEN）→ 重構（IMPROVE）
/tdd
3
遇到建置錯誤用 /build-fix
不用自己看錯誤訊息，讓 build-error-resolver agent 自動修
/build-fix
4
完成後 /code-review 審查
確認沒有安全漏洞或品質問題
/code-review

## 🔴 RED：先寫測試 ```ts // cart.test.ts describe('Cart', () => { it('新增商品後數量應增加', () => { const cart = new Cart(); cart.addItem({ id: '1', name: 'T-Shirt', price: 299 }); expect(cart.items).toHaveLength(1); }); it('同商品重複新增應累加數量', () => { const cart = new Cart(); cart.addItem({ id: '1', name: 'T-Shirt', price: 299 }); cart.addItem({ id: '1', name: 'T-Shirt', price: 299 }); expect(cart.items[0].quantity).toBe(2); }); }); ``` 測試目前會 FAIL ✓（因為還沒有實作） 準備進入 🟢 GREEN 階段？
yes

## 🟢 GREEN：最小實作讓測試通過 ```ts // cart.ts export class Cart { items: CartItem[] = []; addItem(product: Product) { const existing = this.items.find(i => i.id === product.id); if (existing) { existing.quantity += 1; } else { this.items.push({ ...product, quantity: 1 }); } } } ``` ✅ 2/2 測試通過 準備進入 🔵 IMPROVE 重構階段？

## 🔧 Build Error Resolver 偵測到錯誤： ``` TS2345: Argument of type 'string' is not assignable to parameter of type 'number' at cart.ts:15 ``` **問題分析：** price 欄位定義為 number，但傳入了 string "299" **修復方案：** ```ts // 修復前 cart.addItem({ id: '1', price: "299" }) // 修復後 cart.addItem({ id: '1', price: 299 }) ``` ✅ 已修復 1 個 TypeScript 錯誤 ✅ Build 成功
