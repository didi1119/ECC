自動化魔法

讓 Claude 幫你省時間

📖 情境故事

老王每次存檔都要手動跑 prettier，每次 commit 前都要手動跑 lint。 他設定了 ECC 的 Hooks，從此存檔自動格式化、commit 前自動品質檢查、 session 結束自動驗證 build。一週省下了 2 小時重複操作。

什麼是 Hooks？

Hooks 是在特定事件發生時自動執行的指令。不需要你手動觸發，Claude Code 自動幫你跑。

PreToolUse

工具執行前觸發（可以阻止）

例：在 git commit 前先跑 lint

PostToolUse

工具執行後觸發

例：存檔後自動格式化

Stop

每次 Claude 回應結束時觸發

例：Session 結束自動跑 build

最實用的 4 個 Hook 配置

存檔自動格式化PostToolUse: Write/Edit
prettier --write "$FILE_PATH"

✓ 永遠不需要手動跑 prettier

ESLint 自動修復PostToolUse: Write/Edit
eslint --fix "$FILE_PATH"

✓ 即時修復 lint 警告

提交前品質檢查PreToolUse: Bash（git commit）
lint-staged && tsc --noEmit

✓ 阻止有問題的程式碼進入 git

Session 結束建置驗證Stop
npm run build

✓ 每次工作結束確認 build 正常

設定範例

修改 ~/.claude/settings.json 加入 Hooks 設定

~/.claude/settings.json
// ~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx prettier --write \"$FILE_PATH\"",
        "description": "每次存檔自動格式化"
      }
    ]
  }
}

💡 ECC 已在安裝時幫你配置了許多實用的 Hooks，可以直接使用

/loop — 定時重複執行

用 /loop 設定定時任務，例如每 5 分鐘自動跑一次 code review

情境示範 — /loop 定時任務
/loop 5m /code-review
U
C
已設定循環任務 ✅ **每 5 分鐘**自動執行 `/code-review` 目前狀態： - 下次執行：4:58 後 - 已執行次數：0 - 模式：持續運行（直到你停止） 輸入 `/loop-status` 查看狀態 輸入 `stop` 或關閉視窗停止
每 10 分鐘自動審查程式碼
> /loop 10m /code-review
每 30 分鐘自動更新文件
> /loop 30m /docs
