/multi-workflow

⚡自動化

前後端同時推進的完整全棧工作流，智慧分配任務給前後端各自的模型

📖 這個指令在做什麼？

/multi-workflow 是 /multi-backend 和 /multi-frontend 的合體版——完整的全棧功能開發工作流。它會智慧地把前端任務和後端任務分別交給最擅長的 AI 模型處理，然後整合結果。

整個流程涵蓋：需求研究 → 前後端分頭設計 → 前後端平行實作 → 整合測試 → 效能優化 → 程式碼審查。每個階段的輸出自動流入下一個階段。

這是實作一個完整功能最全面的方法，但也是最費時的——適合真正重要的核心功能，而不是小修改。

🕐 什麼時候用？

  • ✓實作一個需要前後端配合的完整功能
  • ✓有充足時間，想要最高品質的全棧實作
  • ✓新功能涉及新的 API 設計和對應的前端 UI
  • ✓需要完整的端到端測試覆蓋

💬 實際輸入範例

$/multi-workflow 實作完整的商品搜尋功能：全文搜尋、篩選、排序、分頁
