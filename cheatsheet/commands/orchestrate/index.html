/orchestrate

⚡自動化

讓一個主 Claude 負責指揮，派出多個子 Claude 平行完成任務

📖 這個指令在做什麼？

在一般的開發流程裡，Claude 是一個人幫你做所有事——先做這個，再做那個，一件一件來。/orchestrate 則是「交響樂團模式」：一個指揮家（主 Claude）負責統籌大局，然後同時派出多個演奏家（子 Claude）各自負責一個部分，同時進行。

就像蓋大樓，一般做法是同一組工人先打地基、再蓋牆、再裝屋頂。但交響樂模式是：地基組、牆面組、屋頂組同時開工，最後指揮確認所有部分組合正確。

這樣的好處是速度可以快 3-5 倍。適合那種「前端和後端可以同時做，不互相等待」的任務。

🕐 什麼時候用？

  • ✓一個大功能有多個完全獨立的子任務
  • ✓前端元件、API 端點、資料庫設計可以同時進行
  • ✓有 10 個以上的頁面需要同時更新
  • ✓需要在最短時間完成大量不相依的工作

💬 實際輸入範例

$/orchestrate 同時完成這三個獨立任務： 1. 建立用戶個人頁面 API（app/api/users/[id]/route.ts） 2. 建立用戶頁面前端元件（app/users/[id]/page.tsx） 3. 更新 README 加入新 API 說明

⚡注意事項 & 小技巧

⚠️任務必須真正獨立才能平行

如果 Agent B 需要等 Agent A 的結果才能開始，那就不是真正獨立。平行執行這種有依賴關係的任務會導致衝突或錯誤。先確認任務彼此不相依。

💡避免多個 Agent 修改同一個檔案

如果兩個 Agent 都要改 app/utils.ts，會產生衝突。/orchestrate 前要確認每個任務的修改範圍不重疊。

✨搭配 /multi-plan 效果最好

先用 /multi-plan 讓 Claude 把大任務拆解成獨立子任務並規劃，確認後再用 /orchestrate 執行，效果最佳。
