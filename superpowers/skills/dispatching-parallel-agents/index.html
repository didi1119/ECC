⚡

superpowers:dispatching-parallel-agents

development

識別獨立問題域，同時派發多個專業子代理並行處理

這個技能在做什麼？

當多個問題或任務彼此獨立時，可以同時派發多個子代理並行處理，大幅縮短總時間。

每個代理都有聚焦的範疇、自包含的提示詞和明確的輸出格式。完成後統一審查結果，確認沒有衝突。

什麼時候用？

  • ✓多個不相關的測試失敗
  • ✓跨模組的平行功能開發
  • ✓調查彼此獨立的問題
  • ✓需要同時進行多個代碼審查

如何啟用？

>use superpowers:dispatching-parallel-agents

執行步驟

1

識別獨立問題域

確認問題之間沒有共享狀態或依賴關係

2

為每個代理建立聚焦任務

每個任務：明確範疇 + 目標 + 限制 + 預期輸出格式

3

同時派發所有代理

並行執行，而非依序等待

Task (parallel): [問題 1] / [問題 2] / [問題 3]
4

審查並整合結果

檢查修復是否衝突，驗證全部測試通過

對話範例

有 6 個測試失敗：3 個在 auth 模組、2 個在 payment 模組、1 個在 notifications

U
C

好的，我來使用 superpowers:dispatching-parallel-agents 技能。 這 3 個問題域彼此獨立，可以並行處理： [同時派發 3 個子代理] → Agent A：auth 模組 3 個失敗 → Agent B：payment 模組 2 個失敗 → Agent C：notifications 1 個失敗 全部完成後整合結果，確認無衝突...

結果如何？

U
C

✅ 全部 6 個測試現在通過： - Agent A 修復：缺少 JWT 過期驗證 - Agent B 修復：非同步處理 race condition - Agent C 修復：錯誤的 email template 路徑 修復無衝突，可以 commit。

核心原則

→只在問題真正獨立時才並行
→每個代理的提示詞必須完全自包含
→必須確認所有修復不會相互衝突
→一個代理一個問題域
