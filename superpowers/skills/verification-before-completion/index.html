SuperpowersVerification Before Completion
✅

superpowers:verification-before-completion

debugging

確認修復解決了根因而非只讓測試通過，並檢查邊緣案例和 regression

這個技能在做什麼？

這個技能防止一個常見的陷阱：測試通過了，但問題其實沒有真正解決（只是測試不夠嚴格）。

它要求在宣告完成之前，明確驗證根因已被解決、邊緣案例都有涵蓋、沒有引入新的 regression。

什麼時候用？

  • ✓宣告 bug 修復完成前
  • ✓每次 git commit 之前
  • ✓systematic-debugging 的最後一步

如何啟用？

>use superpowers:verification-before-completion

執行步驟

1

確認根因已解決

不只是症狀消失，而是根本原因已移除

2

執行完整測試套件

不只是相關測試，是全部測試

npm test
3

檢查邊緣案例

空值、極端值、並發情況都有測試

4

確認無 regression

與此修復相關的所有功能都還正常工作

對話範例

session 問題修復了，測試通過

好的，我來使用 superpowers:verification-before-completion 技能。 確認根因已解決：時區問題已修正，不只是 session 時間延長 ✓ 執行完整測試套件：82/82 通過 ✓ 邊緣案例： - 跨越午夜的 session ✓ - 夏令時間切換 ✓ - 並發 token refresh ✓ 確認無 regression：auth、payment、notifications 模組測試全通過 ✓ 驗證完成，可以安全 commit。

核心原則

→測試通過不等於問題解決
→必須明確驗證根因已移除
→不只跑相關測試，要跑完整套件
→邊緣案例要有測試覆蓋
