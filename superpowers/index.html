安裝指南

三步驟完成安裝，在任何 Claude Code 專案中使用

1

Clone 技能庫

將 Superpowers 技能庫加入你的專案

git clone https://github.com/obra/superpowers vendor/superpowers
2

執行安裝腳本

將技能複製到 Claude Code 全域技能目錄

.\scripts\install-superpowers.ps1# Mac / Linux: bash scripts/install-superpowers.sh
3

驗證安裝

在 Claude Code 中輸入以下指令確認安裝成功

use superpowers:using-superpowers

所有技能

14 個技能，按開發階段分類

📋

規劃

設計優先，避免浪費

3 個技能
💡
superpowers:brainstorming

透過 9 步驟 Socratic 對話將想法精煉為設計文件，再開始實作

📝
superpowers:writing-plans

將規格書拆解成 2-5 分鐘可完成的 TDD 任務清單，帶完整代碼片段

▶️
superpowers:executing-plans

載入計畫檔案，依序執行每個任務，在每個任務後進行驗證

🤖

開發

子代理平行作業

2 個技能
🤖
superpowers:subagent-driven-development

為每個任務派發全新子代理，搭配規格合規 + 代碼品質雙階段審查

⚡
superpowers:dispatching-parallel-agents

識別獨立問題域，同時派發多個專業子代理並行處理

🌿

版控

隔離工作區管理

2 個技能
🌿
superpowers:using-git-worktrees

為每個功能建立隔離的 git worktree，避免分支切換污染工作狀態

🏁
superpowers:finishing-a-development-branch

驗證測試後提供四個選項：本地合併、建立 PR、保留或捨棄

🔍

品質

TDD + 代碼審查

3 個技能
🔴
superpowers:test-driven-development

RED-GREEN-REFACTOR 循環：先寫失敗測試，再寫最小實作，最後重構

📋
superpowers:requesting-code-review

提交前對照規格書的自我審查清單，確認所有需求都實作完整

✉️
superpowers:receiving-code-review

系統化處理審查意見：分類優先級，逐一修復，確認每個問題都解決

🐛

除錯

根因分析四階段

2 個技能
🔍
superpowers:systematic-debugging

四階段根因分析：調查 → 模式比對 → 假設驗證 → 實作，禁止隨機嘗試

✅
superpowers:verification-before-completion

確認修復解決了根因而非只讓測試通過，並檢查邊緣案例和 regression
